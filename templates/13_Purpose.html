Purpose of the Application

  • Currently the Application has two main objectives:
    1. Lay down the first stone in the development of a Free and Open Source Application (such as Gimp) in the Talent Acquisition Market.
    2. Demonstrate my Skills as a Web Developer, mainly in the Back-end (Here is the Github and API Documentation).
  • Regarding point one, the Application works as follows:
    • You either have to register as a freelancer or a company. if you don't want to register, clicking here will take you to a picture displaying the "Find Talent" feature if you want to check it out.
    • If you want to register as a freelancer, you have to consider that:
      • Your Name, Profession, Hourly Rate, Personal Description, GitHub, Linkedin, Instagram, Facebook and Profile Picture are going to be displayed on the "Find Talent" section of the Application.
      • Your Email will be inaccessible from the Application, but Company users will be able to send you proposals by clicking on the "Contact Professional" button.
      • The Categories and Skills you select will be used as filters in the search process.
    • If you want to register as a company, you have to keep in mind that every time you click on the "Contact Professional" button, your Company Name, Personalized Message, Contact Email, Contact Phone, Likedin and Instagram will be sent to the potential candidate.
    • Once you are registered, you can log in and start using the Application, the most important feature that it has is the one called "Find Talent", which allows you to send pre-built emails to potential candidates by clicking the "Contact professional" button if you are a company, otherwise, it only allows you to look at the candidates registered on the platform.
    • A more detailed explanation is being worked on, but due to time constraints it is not yet available, however, the application is a simple, intuitive and easy-to-use solution, so don't be afraid to use it. Please have in consideration that when you create a user the password is hashed.
  • Regarding point two, you can check the API documentation here and the Application documentation here. The technologies used in this project are:
    • FastAPI as the main Web Framework. It is used for all types of tasks, from creating endpoints to server-side rendering.
    • PostgreSQL as the relational database management system. This Application uses SQL for almost everything, you can see, for example, that if you want to send an email to another user, there are several limitations, all these limitations are imposed by PostgreSQL.
    • Pydantic for data validation and settings management using Python-type annotations. Pydantic ensures that the user input matches the specified data types in the Application.
    • Alembic for database migrations. It allows to:
      • Emit ALTER statements to the database in order to change the structure of the tables and other constructs.
      • Have a system whereby “migration scripts” are constructed; each script indicates a particular series of steps that can “upgrade” a target database to a new version, and optionally a series of steps that can “downgrade” similarly, doing the same steps in reverse.
      • Execute the scripts in a sequential manner.
    • Git and GitHub to create, store, manage, share and track changes in the source code of the Application. It also makes the process to push code from the development enviroment to production very easy.
    • An Ubuntu virtual machine hosted by DigitalOcean on which the Application is deployed.
    • Nginx mainly as a reverse proxy. Currently it is only allowing HTTP and HTTPS traffic, limiting the amount of requests per IP per second, establishing connection timeouts and limiting the amount of data you can upload in each request.
    • Certbot to obtain SSL certificates from Let's Encrypt and also to enable HTTPS on the server.
    • Uvicorn as the binding element that handles the web connections from the browser or api client, so FastAPI can serve the actual request. Uvicorn listens on a socket, receives the connection, does a bit processing and hands the request over to FastAPI.
    • Gunicorn as a process manager running multiple Uvicorn worker processes.
    • OpenCV for human face detection using pre-trained machine learning models.
    • Pillow for image processing.
    • JSON Web Tokens to authorize access to the Application resources.
    • Pytest to write API test cases.
    • Jinja as a fast, expressive, extensible templating engine.
    • HTML, CSS and Javascript for Front-End web development. No Javascript frameworks or libraries are currently being used.
    • Brevo as the SMTP server for sending emails.
    • Buy me a coffee as the donation platform.
    • Tableau for Application monitoring and control.
